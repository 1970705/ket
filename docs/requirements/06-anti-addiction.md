# 防沉迷系统需求

**文档版本**: 1.0
**创建日期**: 2026-02-20
**作者**: Wordland团队
**状态**: 设计完成，待实施
**优先级**: P2 (长期需求，1-2月)

---

## 📋 文档概述

本文档详细说明Wordland KET词汇学习应用的防沉迷系统需求，包括会话管理、休息奖励系统、家长控制和COPPA合规要求。

### 核心原则

1. **儿童安全优先**: 防止过度使用屏幕
2. **正向强化**: 奖励健康使用行为
3. **家长赋权**: 提供家长控制权和可见性
4. **最小干扰**: 平滑融入学习流程

## 1. 会话管理需求

### 1.1 会话时间限制

**默认限制** (可由家长配置):

| 限制类型 | 时长 | 触发条件 | 行为描述 | 可配置 |
|---------|-----|----------|----------|--------|
| **软提醒** | 15分钟 | 单次会话计时器 | 温和弹窗提示："要休息一下吗？" | ✅ 是 |
| **硬限制** | 45分钟 | 单次会话计时器 | 强制休息："已经学习45分钟！休息一下吧！" | ✅ 是 |
| **每日上限** | 2小时 | 每日累加器 | 阻塞进入："明天再来学习吧！" | ✅ 是 |
| **休息提醒** | 每5分钟 | 活跃游戏时间间隔 | 微妙通知："远眺20秒！" | ✅ 是 |

### 1.2 活跃时间检测

**仅计算以下时间**:
- ✅ 活跃游戏时间（答题、拼写）
- ✅ 学习界面停留时间

**不计算以下时间**:
- ❌ 应用后台运行
- ❌ 暂停状态
- ❌ 在菜单/设置界面
- ❌ 加载界面
- ❌ 阅读说明/教程

### 1.3 时间阈值

**教育依据** (10岁儿童发展特点):

| 年龄组 | 注意力持续时间 | 推荐学习时长 | 休息间隔 |
|--------|---------------|-------------|---------|
| 10-12岁 | 15-20分钟 | 15分钟/次 | 每5分钟提醒 |

**实现参数**:
```kotlin
const val SOFT_REMINDER_MS = 15 * 60 * 1000L      // 15分钟
const val HARD_LIMIT_MS = 45 * 60 * 1000L         // 45分钟
const val DAILY_MAX_MS = 2 * 60 * 60 * 1000L      // 2小时
const val BREAK_REMINDER_INTERVAL_MS = 5 * 60 * 1000L  // 5分钟
```

## 4. COPPA合规需求

### 4.1 家长同意机制

**同意流程**:
1. 首次启动应用时显示家长同意界面
2. 解释数据收集目的和范围
3. 收集家长邮箱地址
4. 发送验证邮件
5. 验证通过后激活账户

**同意界面需求**:
```
┌────────────────────────────────────┐
│                                    │
│      需要家长同意                   │
├────────────────────────────────────┤
│                                    │
│  要使用Wordland，我们需要您的       │
│  许可收集以下使用数据：             │
│                                    │

│  ✓ 会话时间管理                     │

│  [阅读完整隐私政策]                │
│                                    │
│  家长邮箱：                        │

│                                    │

│                                    │
└────────────────────────────────────┘
```

### 4.2 数据最小化原则

**收集的数据**:
- ✅ 会话时长（学习时间）
- ✅ 学习进度（词汇掌握、关卡完成）
- ✅ 使用模式（活跃时段、频率）
- ✅ 家长邮箱（仅用于报告）

**不收集的数据**:
- ❌ 位置信息
- ❌ 儿童联系方式
- ❌ 音视频录制
- ❌ 社交媒体信息
- ❌ 生物识别数据

### 4.3 退出机制

**家长可以**:
- 随时查看收集的数据
- 导出所有儿童数据
- 删除账户和所有数据
- 撤销数据收集同意

**实现方式**:
- 应用内"家长控制" → "数据管理"
- 邮件发送请求
- 10天内响应

### 4.4 数据保留政策

| 数据类型 | 保留期限 | 说明 |
|---------|---------|------|
| 每日使用数据 | 90天 | 滚动删除 |
| 每周报告 | 1年 | 用于趋势分析 |
| 学习进度 | 直到账户删除 | 用户核心数据 |

CREATE TABLE daily_usage (

CREATE TABLE parental_settings (

- [ ] 会话计时器仅计算活跃游戏时间
- [ ] 软提醒在15分钟时显示
- [ ] 硬限制在45分钟时强制休息
- [ ] 每日上限在2小时时阻止进入
- [ ] 休息奖励正确计算并应用
- [ ] 家长PIN正确保护设置
- [ ] 睡眠模式在设定时段内锁定
- [ ] 护眼提醒每5分钟显示

### 8.2 UX验收

- [ ] 计时器可见且不侵入
- [ ] 休息提醒感觉鼓励而非惩罚
- [ ] 家长仪表板直观易用
- [ ] 儿童能理解休息系统
- [ ] 休息返回感觉有奖励感
- [ ] 每日上限界面正向积极

### 8.3 COPPA合规验收

- [ ] 家长同意机制完整
- [ ] 数据收集最小化
- [ ] 退出机制有效
- [ ] 数据保留政策执行

## 10. 成功指标

### 10.1 健康指标

| 指标 | 目标值 | 测量方式 |
|------|--------|---------|
| 平均会话时长 | 15-30分钟 | 会话追踪 |
| 日活<1小时用户占比 | >80% | 使用分析 |
| 日活>2小时用户占比 | <5% | 使用分析 |
| 休息完成率 | >70% | 休息追踪 |
| 家长控制使用率 | >50% | 功能分析 |

### 10.2 参与度指标

| 指标 | 目标值 | 测量方式 |
|------|--------|---------|
| 7天留存率 | >60% | 群组分析 |
| 30天留存率 | >40% | 群组分析 |
| 日均学词数 | 10-15个 | 进度追踪 |
| 平均连续天数 | >5天 | 连续追踪 |

| 指标 | 目标值 | 测量方式 |
|------|--------|---------|
| 报告打开率 | >70% | 邮件分析 |
