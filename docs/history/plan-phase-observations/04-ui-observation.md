## 3. 任务优先级建议

### P0 (本周必须完成)

#### 1. 优化 LearningScreen 重组性能
**理由**: 432行单文件是性能瓶颈，影响游戏核心体验

**拆分计划**:
```
LearningScreen (主容器)
├── QuestionHeader (题目显示)
├── AnswerSection (答案框 + 进度)
├── KeyboardSection (虚拟键盘)
├── FeedbackOverlay (正确/错误反馈)
└── HintPanel (提示面板)
```

**技术要点**:
- 使用 `@Immutable` 标记数据类
- 提取纯函数组件
- 使用 `key()` 帮助 Compose 跟踪

#### 2. 集成增强 Hint 系统 UI
**理由**: 架构已完成 (HintGenerator, HintManager, UseHintUseCaseEnhanced)，只差 UI 展示层

**UI 需求**:
- 提示按钮 (显示可用次数)
- 提示内容卡片 (根据等级显示不同内容)
- 冷却状态指示
- 使用警告 (次数不足)

#### 3. 实现动态星级评分 UI
**理由**: 当前固定 3 星降低游戏成就感，不公平的评分影响学习动机

**实现要点**:
- 接收后端评分结果
- 星级获得动画 (逐个显示)
- 保存到本地状态
- 与 LevelCompleteScreen 集成

### P1 (重要)

#### 4. 添加 UI 组件测试框架
**理由**: 0% 覆盖率是不可接受的风险

**测试优先级**:
1. SpellBattleGame - 核心游戏交互
2. WordlandButton - 按钮组件
3. WordlandCard - 卡片组件
4. HintCard - 提示系统

#### 5. 增强键盘反馈体验
**理由**: 提升游戏手感，减少误操作

**改进点**:
- 按键按下效果 (颜色变化/缩放)
- 正确答案时键盘高亮
- 错误答案时震动反馈

### P2 (改进)

#### 6. 关卡完成庆祝动画增强
- 更丰富的粒子效果
- 星级飞入动画
- 记忆强度增长动画

#### 7. 添加加载状态和空状态组件
- 骨架屏加载
- 数据为空时的占位图

## 4. 风险识别

### 4.1 技术风险

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 过度重组导致掉帧 | 中 | 高 | 使用 Compose Compiler Report 分析，添加 `stable` 标记 |
| 不同屏幕尺寸适配问题 | 中 | 中 | 使用 `WindowSizeClass` + 响应式布局，多设备预览 |
| 动画在低端设备卡顿 | 高 | 中 | 添加性能降级开关，简化动画 |
| Compose UI 测试学习曲线 | 高 | 低 | 从简单组件开始，参考官方示例 |

### 4.2 用户风险

| 风险 | 缓解措施 | 状态 |
|------|----------|------|
| 10 岁儿童可能误操作 | 最小 48dp 触摸目标，确认对话框 | ✅ 已遵循 |
| 颜色对比度不足 | 遵循 WCAG AA 标准 (4.5:1) | ⚠️ 需验证 |
| 动画可能引起眩晕 | 提供动画开关设置 | ❌ 未实现 |
| 字体太小看不清 | 使用大号字体 (14sp+) | ✅ 已遵循 |

### 4.3 性能风险

| 指标 | 目标 | 当前状态 |
|------|------|----------|
| 帧率 | 60fps | 未验证 |
| 首帧渲染 | <16ms | 未测量 |
| 重组次数 | 最小化 | 需优化 |
| 内存占用 | <100MB | 未测量 |

## 5. 对 Plan Agent 的建议

### 5.1 UI 相关任务应该优先

1. **Hint 系统集成** 是用户可见的高价值功能，应该优先完成
2. **LearningScreen 性能优化** 是技术债务，越早还越好
3. **UI 测试** 应该与功能开发同步进行

### 5.2 设计建议

1. **动画一致性**: 使用统一的动画时长和缓动函数
2. **色彩系统**: 扩展现有配色，为更多岛屿做准备
3. **组件库**: 考虑封装为独立的 UI Kit 模块

### 5.3 需要协作

- 与 **android-engineer** 协作: Hint 系统 API 对接
- 与 **android-performance-expert** 协作: 性能优化验证
