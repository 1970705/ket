## 1. 地图系统重构

### 1.1 当前问题

- ❌ 应用名为"Wordland"(单词大陆)但实际显示孤立岛屿
- ❌ 缺乏探索感和世界感
- ❌ 用户不知道自己在哪，要去哪

### 1.2 设计方案: 混合方案

#### 世界视图 vs 岛屿视图

**世界视图**:
- 显示群岛整体布局
- 显示玩家船只位置
- 显示区域解锁状态
- 迷雾遮盖未解锁区域

**岛屿视图**:
- 显示当前岛屿详情
- 显示关卡布局
- 显示进度和精通度
- 方便关卡选择

### 1.3 迷雾系统

#### 迷雾设计

**视觉效果**:
- 半透明白色渐变 (alpha: 0.6 → 0.9)
- 边缘模糊 (10px blur)
- 未解锁区域完全遮盖

**可视半径**:
```kotlin
fun calculateVisibilityRadius(playerLevel: Int): Float {
    return when {
        playerLevel <= 3 -> 0.15f   // 地图的15%
        playerLevel <= 6 -> 0.30f   // 地图的30%
        else -> 0.50f               // 地图的50%
    }
}
```

**迷雾揭开动画**:
- 持续时间: 500ms
- 缓动函数: easeOut
- 视觉效果: 径向渐变消失

### 1.4 玩家船只

#### 船只设计

**图标**: 🚢 (或自定义船只图标)

**位置显示**:
- 在世界视图显示当前区域
- 平滑移动动画 (300ms)
- 位置标记: 蓝色圆点 + 脉冲动画

**移动动画**:
```
起点区域 → 中间过渡 (500ms) → 终点区域
         ↓
    缓动函数: easeInOut
```

## 2. 视觉反馈增强

### 2.1 分层反馈系统

#### 层级1: 即时反馈 (0-100ms)

**正确反馈**:
- 颜色: 绿色 (#4CAF50)
- 动画: 字母框淡入淡出 (100ms)
- 触觉: 短脉冲震动 (50ms)
- 音效: 清脆"叮"声 (200ms)

**错误反馈**:
- 颜色: 红色 (#F44336)
- 动画: 字母框抖动 (100ms)
- 触觉: 无 (避免负面感受)
- 音效: 温和"唔"声 (300ms)

#### 层级2: 详细反馈 (100-500ms)

**星级评分动画**:
- 星星依次弹出 (每个200ms)
- 缩放动画 (0.8 → 1.2 → 1.0)
- 颜色渐变 (灰色 → 金色)

**记忆强度条动画**:
- 平滑过渡 (500ms)
- 颜色变化 (红 → 黄 → 绿)
- 数字滚动效果

#### 层级3: 庆祝效果 (里程碑)

**3星完成**:
- 全屏彩带粒子
- 庆祝音效 (1500ms)
- 文字: "Perfect!" / "太棒了!"

**5连击**:
- 屏幕微震 (5px)
- 火焰图标放大
- 文字: "On Fire!" / "势不可挡!"

**关卡完成**:
- 全屏庆祝动画
- 宠物跳跃动画
- 进度保存提示

### 2.2 宠物反应系统

#### 宠物状态

```kotlin
enum class PetReaction {
    // 正向反应
    ECSTATIC,      // 3星,无提示 (跳跃 + 欢呼动画)
    PROUD,         // 2星且有进步 (点头 + 竖大拇指)
    ENCOURAGING,   // 1星或错误但有努力 (微笑 + 鼓励手势)
    CELEBRATING,   // 关卡完成 (旋转 + 烟花)
    COMBO_HIGH,    // 5+连击 (兴奋摇摆)

    // 负向反应 (温和,不打击)
    CONCERNED,     // 使用提示后错误 (歪头 + 思考)
    SURPRISED,     // 检测到猜测 (眨眼 + 问号)
    SLEEPY,        // 多次快速错误 (打哈欠 + 揉眼睛)
}
```

#### 宠物动画

**位置**: 屏幕右下角

**大小**: 64dp × 64dp

**动画时长**: 500-1000ms

## 5. 儿童友好设计

### 5.1 易用性原则

#### 字体和文字

- **字体大小**: 最小16sp
- **行高**: 1.5倍字体大小
- **颜色对比度**: ≥ 4.5:1 (WCAG AA标准)
- **避免**: 全大写文字 (难以阅读)

#### 按钮和交互

- **最小点击区域**: 48dp × 48dp
- **按钮间距**: ≥ 16dp
- **圆角**: 8dp - 12dp
- **阴影**: 轻微提升层次感

#### 图标和图像

- **简单明了**: 避免复杂细节
- **一致性**: 相同功能相同图标
- **色彩**: 丰富但不过度
- **动画**: 流畅不晕眩

### 5.2 情感化设计

#### 鼓励性文案

| 场景 | 文案 |
|------|------|
| 首次解锁成就 | "太棒了！你解锁了第一个成就！" |
| 进度更新 | "继续努力，还差一点点！" |
| 长期目标 | "这是一个长期目标，慢慢来，不着急！" |
| 成就解锁失败 | "没关系，下次一定能成功！" |

#### 色彩心理学

| 颜色 | 用途 | 心理效果 |
|------|------|----------|
| 蓝色 | 主色调 | 平静、信任 |
| 绿色 | 成功、正确 | 积极、成长 |
| 黄色 | 警告、注意 | 温和、友好 |
| 橙色 | 奖励、庆祝 | 热情、兴奋 |
| 红色 | 错误 (柔和) | 不严厉 |

## 6. 动画和过渡

### 6.1 动画原则

#### Material Design Motion

**Duration**:
- 微交互: 100ms
- 常规交互: 200-300ms
- 复杂交互: 400-500ms
- 场景转换: 500-600ms

**Easing**:
- 进入: easeOut (快速进入,缓慢停止)
- 退出: easeIn (缓慢开始,快速退出)
- 持续: easeInOut (两端缓慢,中间快速)

#### 性能要求

- **目标帧率**: 60fps
- **最大帧时间**: 16ms
- **避免**: 过度绘制、复杂路径、大量透明度

### 6.2 过渡动画

#### 页面切换

```
A页面 → B页面
  ↓
淡出A (200ms) + 滑入B (200ms)
  ↓
同时进行 (总200ms)
  ↓
easeOut曲线
```

#### 状态变化

```
旧状态 → 新状态
  ↓
旧状态淡出 (100ms)
  ↓
新状态淡入 (100ms)
  ↓
轻微延迟 (50ms)
  ↓
总时长: 250ms
```
