# Quick Judge 游戏机制设计文档

## 1. 游戏概述

### 1.1 游戏目标

Quick Judge（快速判断）是 Wordland 中一款速度与准确度并重的词汇学习游戏模式。玩家需要在有限时间内判断屏幕上显示的中文翻译是否与英文单词匹配。

**核心目标**：
- 训练词汇识别速度
- 提高词汇记忆准确度
- 通过快速反馈建立学习心流

### 1.2 核心玩法

```
1. 屏幕显示：英文单词 + 中文翻译
2. 玩家判断：翻译是否正确？
3. 快速选择：点击"正确"或"错误"按钮
4. 即时反馈：显示判断结果 + 得分
5. 下一题：继续挑战或完成关卡
```

**游戏特点**：
- **速度挑战**：倒计时机制，考验反应速度
- **连击系统**：连续正确答对获得奖励加成
- **难度递进**：Easy/Normal/Hard 三档难度
- **星级评分**：基于准确率和速度的动态评分

### 1.3 目标用户

| 用户特征 | 描述 |
|---------|-----|
| **年龄** | 10岁儿童（小学4-6年级） |
| **英语水平** | KET 初级（A2水平） |
| **认知特点** | 注意力持续时间10-15分钟，喜欢即时反馈 |
| **游戏经验** | 熟悉手机操作，需要简单易懂的UI |

## 2. 难度系统

### 2.1 难度分级

| 难度 | 关卡类型 | 时间限制 | 题目数量 | 最大错误 | 目标用户 | 描述 |
|------|---------|---------|---------|---------|---------|-----|
| **Easy** | 教学关卡 | 10秒/题 | 5题 | 2次 | 初学者 | 温和入门，建立信心 |
| **Normal** | 标准关卡 | 8秒/题 | 10题 | 2次 | 已掌握基础 | 标准挑战，保持心流 |
| **Hard** | 挑战关卡 | 5秒/题 | 15题 | 1次 | 高水平玩家 | 高难度，刺激肾上腺素 |

### 2.2 自适应难度

**初始难度设置**：
- 首次进入游戏：默认 **Easy** 模式
- 用户可手动选择任意难度

**自动升级条件**：
```
连续3关获得3星 → 自动提升难度
示例：Easy → Normal → Hard
```

**自动降级条件**：
```
连续2关获得<2星 → 自动降低难度
示例：Hard → Normal → Easy
```

**手动选择**：
- 玩家随时可在关卡选择界面切换难度
- 切换难度不影响当前进度

### 2.3 难度调整触发时机

| 时机 | 动作 |
|------|-----|
| **游戏开始前** | 显示难度选择器 |
| **关卡完成后** | 根据表现建议难度调整 |
| **连续失败后** | 自动降级并显示鼓励信息 |
| **连续成功后** | 显示升级提示动画 |

### 2.4 时间限制设计理由

| 时间 | 理由 | 用户体验 |
|------|------|---------|
| **10秒** | 给予初学者足够思考时间，减少焦虑 | "我可以慢慢思考" |
| **8秒** | 适中压力，保持心流状态 | "有点紧张但能应对" |
| **5秒** | 挑战反应速度，刺激肾上腺素 | "必须快速反应！" |

**儿童认知参考**：
- 10岁儿童平均词汇识别时间：2-4秒
- 加上判断和点击操作：3-6秒
- 8秒限制给予充足时间但保持紧迫感
- 5秒限制迫使建立直觉反应

## 3. 连击系统

### 3.1 连击定义

**Combo（连击）**：连续正确答题且不中断的次数

**连击条件**：
- 答案正确
- 在时间限制内完成
- 没有使用提示（Quick Judge 无提示功能）

**连击计数规则**：
```
答对 → Combo +1
答错 → Combo 归零
超时 → Combo 归零
```

### 3.2 连击奖励

| 连击数 | 奖励倍数 | 视觉效果 | 音效 | UI提示 |
|--------|---------|---------|------|--------|
| **1-2** | 1.0x（无奖励） | 无 | 无 | 无 |
| **3-4** | 1.2x（+20%） | 🔥 单火焰 + 脉冲动画 | "Combo!" 人声 | "Nice streak!" |
| **5-9** | 1.5x（+50%） | 🔥🔥 双火焰 + 屏幕微震 | "Great!" 人声 | "On fire!" |
| **10+** | 2.0x（+100%） | 🔥🔥🔥 三火焰 + 粒子特效 | "Amazing!" 人声 | "Legendary!" |

**分数计算公式**：
```
基础分 = 100分（答对）/ -20分（答错）
时间奖励 = (剩余时间 / 总时间) × 50分
连击加成 = 基础分 × (连击倍率 - 1.0)

总分 = 基础分 + 时间奖励 + 连击加成
```

**示例计算**：
```
场景：5连击，3秒内答对8秒题
基础分 = 100分
时间奖励 = (5/8) × 50 = 31.25 ≈ 31分
连击加成 = 100 × (1.5 - 1.0) = 50分
总分 = 100 + 31 + 50 = 181分
```

### 3.3 连击中断

**中断条件**：
1. **答错题目** - 无论选择"正确"还是"错误"，只要判断错误
2. **超时未答** - 倒计时结束自动判为错误
3. **退出关卡** - 主动退出或应用关闭

**中断反馈**：
- 视觉：火焰图标熄灭动画
- 音效："Combo中断"低沉音效
- UI：显示"Combo归零"提示1秒

### 3.4 连击恢复

**恢复机制**：
- 连击归零后重新开始计数
- 不保留之前的连击记录
- 但关卡最高连击数会被记录用于统计

**设计理由**：
- 保持游戏紧张感
- 鼓励一气呵成完成关卡
- 避免"分段连击"的策略性行为

### 3.5 连击里程碑

**里程碑定义**：3、5、10连击

**里程碑奖励**：
- **3连击**：首次触发，显示连击系统教程
- **5连击**：触发宠物兴奋动画
- **10连击**：触发全屏庆祝特效

**里程碑持久化**：
- 记录玩家的历史最高连击
- 在成就系统中提供连击相关成就
```
- "初出茅庐"：达成3连击
- "连击大师"：达成10连击
- "神级连击"：达成20连击
```

## 4. 错误惩罚

### 4.1 错误类型

| 错误类型 | 描述 | 触发条件 |
|---------|-----|---------|
| **选择错误** | 玩家判断错误 | 点击"正确"但翻译错误，或反之 |
| **超时错误** | 时间耗尽 | 倒计时归零时未做出选择 |
| **主动放弃** | 玩家退出 | 关卡进行中主动返回 |

### 4.2 惩罚机制

**分数惩罚**：
```
选择错误：-20分
超时错误：-20分（同等处理）
主动放弃：不计分，退出关卡
```

**连击惩罚**：
```
任何错误 → 连击立即归零
无缓冲或"第二次机会"机制
```

**星级惩罚**：
```
每次错误都会影响最终星级评分
3星条件：无错误或最多1次错误（视难度而定）
2星条件：错误次数 ≤ 最大错误次数
1星条件：错误次数 < 最大错误次数 + 2
```

**时间惩罚（可选，暂不实现）**：
```
未来可考虑：答错扣除下一题3秒时间
增加紧张感，但不至于过于严苛
```

### 4.3 失败条件

**关卡失败标准**：
```
Easy（教学）：错误 > 2次 → 失败
Normal（标准）：错误 > 2次 → 失败
Hard（挑战）：错误 > 1次 → 失败
```

**失败后处理**：
1. 显示失败界面（非惩罚性）
2. 展示"再试一次"按钮
3. 显示鼓励性文字：
   - "别放弃，你可以的！"
   - "每次尝试都是进步！"
   - "再试一次，一定行！"
4. 不扣除任何虚拟货币或奖励

**失败后难度调整**：
```
连续失败2次 → 自动降低难度
显示："我们降低了难度，再试一次吧！"
```

### 4.4 惩罚机制设计理由

| 惩罚 | 理由 | 儿童友好考量 |
|------|------|-------------|
| **-20分** | 轻微惩罚，不影响总分太多 | 避免打击信心 |
| **连击归零** | 清晰反馈，鼓励专注 | 重头再来，无负担 |
| **最大错误次数** | 给予容错空间 | 允许偶尔失误 |
| **无负面音效** | 避免挫败感 | 用鼓励代替批评 |

## 5. 星级评分

### 5.1 评分标准

#### Easy 模式评分

| 星级 | 准确率 | 平均时间 | 错误次数 |
|------|--------|---------|---------|
| ⭐⭐⭐ 3星 | ≥ 90% | < 7秒 | ≤ 1次 |
| ⭐⭐ 2星 | ≥ 70% | < 8秒 | ≤ 2次 |
| ⭐ 1星 | ≥ 50% | 任意 | 通过 |
| ✗ 0星 | < 50% | - | 失败 |

#### Normal 模式评分

| 星级 | 准确率 | 平均时间 | 错误次数 |
|------|--------|---------|---------|
| ⭐⭐⭐ 3星 | ≥ 90% | < 5秒 | ≤ 1次 |
| ⭐⭐ 2星 | ≥ 75% | < 6秒 | ≤ 2次 |
| ⭐ 1星 | ≥ 60% | 任意 | 通过 |
| ✗ 0星 | < 60% | - | 失败 |

#### Hard 模式评分

| 星级 | 准确率 | 平均时间 | 错误次数 |
|------|--------|---------|---------|
| ⭐⭐⭐ 3星 | ≥ 95% | < 3秒 | 0次 |
| ⭐⭐ 2星 | ≥ 80% | < 4秒 | ≤ 1次 |
| ⭐ 1星 | ≥ 70% | 任意 | 通过 |
| ✗ 0星 | < 70% | - | 失败 |

### 5.2 星级影响

**进度解锁**：
- 3星：解锁下一难度
- 2星：保持当前难度，可进入下一关
- 1星：建议重玩本关巩固

**奖励系统**：
```
3星：金币奖励 × 1.5，经验值 × 1.5
2星：金币奖励 × 1.0，经验值 × 1.0
1星：金币奖励 × 0.5，经验值 × 0.5
```

**成就系统**：
- "完美主义"：任意关卡获得3星
- "速战速决"：Hard模式获得3星
- "全星收集"：所有关卡获得3星

### 5.3 评分算法

**基础算法**：
```kotlin
fun calculateStars(
    correctCount: Int,
    totalCount: Int,
    avgTimeTaken: Long,
    maxTimeLimit: Int,
    maxErrors: Int,
    errorCount: Int
): Int {
    val accuracy = correctCount.toFloat() / totalCount.toFloat()
    val avgTimeRatio = avgTimeTaken.toFloat() / (maxTimeLimit * 1000f)

    // 先检查是否失败
    if (errorCount > maxErrors) return 0

    return when {
        accuracy >= 0.9f && avgTimeRatio < 0.7f && errorCount <= 1 -> 3
        accuracy >= 0.75f && avgTimeRatio < 0.85f && errorCount <= maxErrors -> 2
        accuracy >= 0.6f -> 1
        else -> 0
    }
}
```

**当前实现状态**：
- ✅ 基础评分算法已实现（`SubmitQuickJudgeAnswerUseCase.calculateStars()`）
- ⏳ 需要优化：加入错误次数考虑
- ⏳ 需要差异化：不同难度不同标准

## 6. 数值平衡

### 6.1 时间限制

| 难度 | 每题时间 | 5题总时间 | 10题总时间 | 15题总时间 |
|------|---------|----------|-----------|-----------|
| Easy | 10秒 | 50秒 | 1分40秒 | 2分30秒 |
| Normal | 8秒 | 40秒 | 1分20秒 | 2分 |
| Hard | 5秒 | 25秒 | 50秒 | 1分15秒 |

**设计考量**：
- Easy 模式总时长控制在 2-3 分钟内
- Normal 模式总时长控制在 1-2 分钟内
- Hard 模式总时长控制在 1 分钟左右
- 符合儿童注意力持续时间特点

### 6.2 分数计算

**分数组成**：
```
总分 = 基础分 + 时间奖励 + 连击加成
```

**基础分**：
```
答对：+100分
答错：-20分
超时：-20分（等同答错）
```

**时间奖励**：
```
公式：(剩余时间 / 总时间) × 50分

示例（10秒题）：
- 1秒完成：9/10 × 50 = 45分
- 5秒完成：5/10 × 50 = 25分
- 9秒完成：1/10 × 50 = 5分
- 超时：0分
```

**连击加成**：
```
3-4连击：基础分 × 0.2
5-9连击：基础分 × 0.5
10+连击：基础分 × 1.0
```

**总分示例**：
```
场景：5连击，2秒完成10秒题
基础分 = 100分
时间奖励 = (8/10) × 50 = 40分
连击加成 = 100 × 0.5 = 50分
总分 = 190分
```

### 6.3 奖励机制

**金币奖励**：
```
基础金币 = 10金币
金币奖励 = 基础金币 × 星级倍率 × 关卡系数

关卡系数：
- Easy: ×1.0
- Normal: ×1.5
- Hard: ×2.0

示例：
- Easy 3星：10 × 1.5 × 1.0 = 15金币
- Normal 3星：10 × 1.5 × 1.5 = 22.5 ≈ 23金币
- Hard 3星：10 × 1.5 × 2.0 = 30金币
```

**经验值奖励**：
```
基础经验 = 20 XP
经验奖励 = 基础经验 × 星级倍率

示例：
- 3星：20 × 1.5 = 30 XP
- 2星：20 × 1.0 = 20 XP
- 1星：20 × 0.5 = 10 XP
```

### 6.4 平衡验证

**测试场景**：

| 场景 | 难度 | 正确率 | 平均时间 | 连击 | 预期星级 |
|------|------|--------|---------|------|---------|
| 初学者首次游玩 | Easy | 60% | 8秒 | 0-2 | ⭐ 1星 |
| 熟练玩家正常发挥 | Normal | 85% | 4秒 | 3-5 | ⭐⭐ 2星 |
| 高手玩家全力挑战 | Hard | 95% | 2秒 | 10+ | ⭐⭐⭐ 3星 |

**平衡调整**：
- 如果 Easy 模式获得3星比例 < 30% → 延长时间到12秒
- 如果 Hard 模式获得3星比例 > 20% → 缩短时间到4秒
- 目标：各难度星级分布符合正态分布

### 6.5 数值总结表

| 参数 | Easy | Normal | Hard |
|------|------|--------|------|
| 时间限制（秒/题） | 10 | 8 | 5 |
| 题目数量 | 5 | 10 | 15 |

## 8. 设计决策

### 8.1 难度曲线设计理由

**递进式设计**：
```
Easy (10s) → Normal (8s) → Hard (5s)
   ↓             ↓              ↓
建立信心    保持心流    挑战极限
```

**设计依据**：
1. **认知负荷理论**：新手需要更多处理时间
2. **心流理论**：匹配技能水平的挑战最投入
3. **自我效能理论**：早期成功建立后续动力

**儿童友好考量**：
- 10秒足够时间减少焦虑
- 渐进式提升培养适应能力
- 自动调整避免挫败感

### 8.2 连击系统设计理由

**奖励而非惩罚**：
- 连击是额外奖励，不是基础分
- 归零后可以立即重新开始
- 不影响游戏基本乐趣

**视觉效果优先**：
- 🔥 火焰图标直观易懂
- 脉冲动画吸引注意但不干扰
- 粒子特效增强成就感

**防沉迷考虑**：
- 连击奖励不影响关卡通过
- 不设计"必须达成X连击"的要求
- 保持轻松愉快的游戏氛围

### 8.3 惩罚机制设计理由

**轻微惩罚原则**：
- -20分不会造成严重损失
- 一次错误不会导致关卡失败
- 保持鼓励性而非惩罚性

**容错空间**：
- Easy/Normal 允许2次错误
- Hard 允许1次错误（挑战性）
- 符合"小步快跑"理念

**正向反馈为主**：
- 答对奖励 > 答错惩罚
- 失败界面强调鼓励而非批评
- "再试一次"而非"游戏结束"

### 8.4 与教育目标平衡

**游戏性 vs 教育性**：

| 设计决策 | 游戏性理由 | 教育性考量 |
|---------|-----------|-----------|
| 快速判断 | 刺激紧张感 | 训练识别速度 |
| 连击系统 | 增加爽快感 | 鼓励连续学习 |
| 时间限制 | 制造挑战 | 提高反应速度 |
| 错误惩罚 | 增加风险 | 强调准确性 |

**学习效果最大化**：
- 速度训练提高词汇自动化识别
- 即时反馈强化正确记忆
- 连击鼓励保持专注状态
- 星级评分给予成就感

**防止不良影响**：
- 时间限制防止过度思考（有利于直觉反应）
- 连击中断防止机械化答题
- 错误反馈确保学到正确答案
