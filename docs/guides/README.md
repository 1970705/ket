## 📝 指南编写规范

### 结构
每个指南应包含：
1. **概述** - 这个指南做什么
2. **前置条件** - 需要什么准备
3. **步骤** - 详细的操作步骤
4. **验证** - 如何验证成功
5. **故障排除** - 常见问题和解决方法

### 命名
```
{TOPIC}_GUIDE.md

示例:
- DEVICE_TESTING_GUIDE.md
- CRASH_DIAGNOSIS_GUIDE.md
```
