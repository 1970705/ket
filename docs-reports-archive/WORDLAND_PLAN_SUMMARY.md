## 📋 核心设计决策总结

### 游戏系统
```
双岛屿渐进解锁
├─ Look Island（60词）- 已解锁
└─ Move Valley（60词）- 需60%精通度解锁
└─ Say City（前60词）- 为下一版准备
```

### 教育系统
```
三级防蒙系统
├─ 行为分析（响应时间<2秒）
├─ 跨场景验证（不同形态考同一词）
└─ 记忆强度模型（动态复习间隔）
```

### 技术栈
```
Kotlin + Jetpack Compose + SQLite + Room
├─ Native Android（先Android，后可考虑iOS）
├─ 纯本地优先（MVP阶段）
├─ 可扩展架构（为云同步预留接口）
```

### 验证方式
```
前后对比（不做A/B测试）
├─ 使用前：KET模拟考基准分
├─ 使用2周后：KET模拟考对比分数
├─ 应用能力：能否在句子中正确使用
└─ 参与度：次日留存、单次时长
```
