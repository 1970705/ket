## 一、产品概述

### 1.1 产品定位

**一句话定位**：
> 让10岁孩子在探索冒险中掌握KET/PET词汇的游戏化学习应用

**产品类型**：
- 教育游戏（Edutainment）
- 词汇学习系统
- KET/PET备考工具

### 1.2 核心价值

| 对象 | 痛点 | 解决方案 |
|------|------|----------|
| 孩子 | 背单词枯燥、记不住、不想学 | 像玩游戏一样学习，不知不觉记单词 |
| 家长 | 担心考试、无法监督学习效果 | 清晰的学习报告、可见的进步轨迹 |
| 考试 | 词汇量不足、不会灵活运用 | 覆盖考纲词表、训练实际应用能力 |

### 1.3 与竞品差异化

| 竞品类型 | 代表产品 | 我们的优势 |
|---------|---------|-----------|
| 纯背词App | 百词斩、墨墨 | 游戏化程度高，不枯燥 |
| 教学App | VIPKid、猿辅导 | 无需预约，随时可学，成本低 |
| 词根产品 | 词根词缀字典 | 10岁友好，游戏化呈现 |
| 教育游戏 | Duolingo | 深度游戏化，而非练习题游戏化 |

## 二、用户分析

### 2.1 主要用户（孩子）

**画像：小明（10岁）**

```
基本信息：
  - 年龄：10岁，四年级
  - 英语基础：学过2年，KET准备中
  - 特点：爱玩游戏（我的世界、Roblox）

行为特征：
  ✓ 喜欢互动、探索、收集
  ✓ 讨厌长文字、说教
  ✓ 需要即时反馈和奖励

痛点：
  ✗ 背单词像受刑
  ✗ 今天背明天忘
  ✗ 考试时想不起来
  ✗ 被父母逼着学

  → "我想玩，不想学"
  → "但我不想让爸妈失望"
```

**画像：妈妈（38岁）**

```
目标：
  - 孩子通过KET/PET考试

  - 性价比合理

关注点：
  ✓ 学了什么？
  ✓ 有没有效果？
  ✓ 孩子是不是在玩而不是学习？

决策因素：
  → 免费试用体验
```

## 三、核心设计理念

### 3.1 设计哲学：游戏为先，学习为辅

```
错误示范：学 + 游戏皮肤 = 孩子不买账
正确方向：好玩的游戏 + 植入学习 = 玩中学
```

**核心原则**：
1. **先好玩**：去掉所有学习元素，游戏本身要好玩
2. **再学习**：学习内容自然融入游戏机制
3. **有效果**：设计确保学习效果可验证

### 3.2 改进后的词根定位

**v1.0问题**：词根作为核心系统对10岁孩子过于抽象

**v2.0方案**：三阶段渐进式引入

```
┌─────────────────────────────────────────────────────────┐
│ 阶段1：语境记忆期（前50-100词）                          │
│   └── 通过场景、句子、声音记忆                            │
│   └── 不提词根概念                                       │
│   └── 目标：建立兴趣和基础感                              │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│ 阶段2：词根萌芽期（100-300词）                            │
│   └── 偶尔出现"词根彩蛋"                                  │
│   └── "哇！这两个词长得好像！"                             │
│   └── 目标：培养词根意识                                   │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│ 阶段3：词根系统期（300+词）                               │
│   └── 正式解锁"词根超能力"系统                            │
│   └── 帮助推断生词、扩展词汇                              │
│   └── 目标：掌握学习工具                                  │
└─────────────────────────────────────────────────────────┘
```

### 3.3 学习科学应用

| 科学原理 | 应用方式 |
|---------|---------|
| **提取练习** | 测试不是检验，而是学习。错误回忆比重复阅读效果好3倍 |
| **间隔重复** | 根据遗忘曲线自动安排复习（1天→3天→7天→15天） |
| **双重编码** | 视觉+听觉+语义三重记忆编码 |
| **情境学习** | 单词在场景中出现，而非孤立词表 |
| **即时反馈** | 3秒内反馈，增强记忆痕迹 |

## 四、游戏玩法系统

### 4.1 世界观：Wordland 词根大陆

**故事背景**：
> Wordland是一个由"语言能量"驱动的神奇世界。每个区域代表一种核心能力（看、说、动、想...）。
>
> 但世界被"遗忘迷雾"慢慢侵蚀。玩家作为"语言守护者"，需要通过掌握语言力量来驱散迷雾，拯救各个区域。

**核心隐喻**：
- 迷雾 = 遗忘
- 词语力量 = 记忆
- 驱散迷雾 = 掌握词汇
- Boss关卡 = 综合测试

### 4.2 四大玩法循环

#### 玩法1：听音寻宝（听力训练）

```
玩法描述：
  └── 听到单词发音 → 在场景中找到对应物体

示例：
  └── 音频："Find the cat!"
  └── 场景：有多个动物的房间
  └── 操作：点击正确的猫

难度曲线：
  ├─ 初级：3个选项，清晰发音
  ├─ 中级：5个选项，加入混淆词
  └─ 高级：7个选项，有句子语境（"Show me the black cat"）
```

#### 玩法2：词句配对（应用训练）

```
玩法描述：
  └── 将单词拖入句子空位，使句子完整

示例：
  └── 句子："I ___ my bag but couldn't ___ it."
  └── 选项：look for / find / look at
  └── 操作：依次拖入正确位置

特点：
  ✓ 训练实际应用能力
  ✓ 符合KET完形填空题型
  ✓ 错误后给予提示而非直接告知
```

#### 玩法3：拼写战斗（拼写训练）

```
玩法描述：
  └── 拼写正确单词来攻击怪物

示例：
  └── 怪物出现
  └── 提示："Look at the TV!"
  └── 玩家按顺序点击字母：W-A-T-C-H
  └── 成功→释放技能攻击；失败→怪物反击

辅助系统：
  ├─ 首字母提示（消耗能量）
  ├─ 元音标记（高级功能）
  └─ 错误位置震动反馈
```

#### 玩法4：快速判断（反应训练）

```
玩法描述：
  └── 快速判断场景和单词是否匹配

示例：
  └── 场景：足球比赛播放中
  └── 出现单词：WATCH
  └── 判断：✓ 正确 → 得分
  └─ 或：LOOK
  └─ 判断：✗ 错误 → 扣生命值

特点：
  ✓ 训练快速反应
  ✓ 适合碎片时间
  ✓ 竞技模式可多人PK
```

### 4.3 关卡结构设计

```
单个世界（约60分钟内容）：
  │
  ├─ 教学关（5分钟）
  │   └─ 介绍核心词，示范用法
  │
  ├─ 5个普通关卡（每个6分钟）
  │   ├─ 交替使用4种玩法
  │   ├─ 每关学习3-4个新词
  │   └─ 复习之前的词汇
  │
  ├─ 小Boss关卡（8分钟）
  │   └─ 综合测试当前世界的词
  │
  ├─ 复习关卡（第二天触发，5分钟）
  │   └─ 不给提示，测试记忆保持
  │
  └─ 大Boss关卡（10分钟）
      └─ 综合测试+引入下一世界预告
```

### 4.4 反馈系统设计

**三级提示机制**：

```
第1次错误：
  └── 引导式提示："再想想，它是动起来的吗？"

第2次错误：
  └── 半帮助提示："你看，足球比赛是在持续变化的..."

第3次错误：
  └── 正确答案 + 解释："正确答案是watch。
                       因为watch是看一段时间，球赛在动"
  └── 立刻给一个简单题重建信心
```

**奖励系统**：

| 奖励类型 | 触发条件 | 效果 |
|---------|---------|------|
| ⭐星星 | 关卡完成 | 解锁新关卡的条件 |
| 🔋能量 | 连续正确 | 可兑换提示 |
| 🎴词卡 | 首次掌握词 | 收集图鉴 |
| 🏆徽章 | 里程碑成就 | 社交炫耀 |
| 🎁宝箱 | 复习关卡全对 | 随机奖励 |

## 五、内容架构

### 5.1 词汇分层体系

```
┌────────────────────────────────────────────────────────┐
│                    KET/PET词汇体系                      │
└────────────────────────────────────────────────────────┘
                          │
        ┌─────────────────┼─────────────────┐
        ↓                 ↓                 ↓
   核心词汇层        拓展词汇层        进阶词汇层
   (必须掌握)       (理解即可)       (PET阶段)
        │                 │                 │
```

**核心词汇层**（MVP阶段优先）：
- KET考试高频词
- 日常生活必备
- 词语构成规律明显

### 5.2 语义地图（8大主题）

```
主题1：LOOK（看） - 看见世界
├─ 核心：look, see, watch, find
├─ 短语：look at, look for, look after, look forward to
└─ 拓展：notice, watch, stare, gaze (PET)

主题2：MOVE（动） - 探索世界
├─ 核心：go, come, move, run, walk, jump
├─ 短语：go to, come back, run away, move on
└─ 拓展：approach, retreat, chase, wander (PET)

主题3：SAY（说）- 表达自我
├─ 核心：say, tell, ask, speak, talk
├─ 短语：ask for, talk to, tell about, speak up
└─ 拓展：explain, describe, discuss, mention (PET)

主题4：FEEL（感）- 理解情绪
├─ 核心：like, love, feel, happy, sad, angry
├─ 短语：feel good, feel like, be happy with
└─ 拓展：excited, nervous, surprised, bored (PET)

主题5：TIME（时）- 把握时间
├─ 核心：day, night, today, tomorrow, yesterday, now
├─ 短语：every day, last night, next week
└─ 拓展：recently, already, yet, still (PET)

主题6：PLACE（地）- 定位空间
├─ 核心：home, school, park, shop, room, place
├─ 短语：at home, in the room, go to school
└─ 拓展：area, region, location, position (PET)

主题7：PEOPLE（人）- 认识他人
├─ 核心：friend, family, teacher, student, child, people
├─ 短语：my friend, family member
└─ 拓展：stranger, neighbor, relative, visitor (PET)

主题8：DO（做）- 付诸行动
├─ 核心：do, make, help, work, play, study
├─ 短语：do homework, make food, play games
└─ 拓展：organize, prepare, achieve, succeed (PET)
```

### 5.3 词根超能力系统（阶段3解锁）

**设计理念**：词根不是"教学内容"，而是"游戏技能"

```
解锁条件：
  └── 完成前3个主题，掌握150+词汇

系统界面：
  └── 看起来像游戏技能树，而非词根表

示例 - VIS超能力：
  📌 技能名：视界之力（VIS Force）
  📌 描述："古老的拉丁语词根vis，意为'看'和'力量'"
  📌 包含词汇：
      ├─ vision（愿景）
      ├─ visit（参观）
      ├─ visual（视觉的）
      ├─ television（电视）
      └─ supervise（监督）
  📌 游戏效果：
      └─ 解锁后，遇到包含vis的陌生词时，
          系统高亮该部分并提示"这是视界之力！"
```

## 六、记忆与学习系统

### 6.1 自适应学习算法

**记忆强度模型**：

```
每个词汇有两个参数：
  ├─ 记忆强度（Memory Strength）：0-100
  └─ 上次复习时间（Last Review）

计算公式（简化版）：
  新强度 = 当前强度 + (正确 × 10) - (错误 × 15)
  复习间隔 = 新强度 ÷ 10（天）

示例：
  ├─ 第1次学：强度=10，次日复习
  ├─ 答对：强度=20，2天后复习
  ├─ 答对：强度=30，3天后复习
  └─ 答错：强度=15，次日复习
```

**智能复习队列**：

```
每日复习词汇来源：
  ├─ 即将遗忘（算法预测）
  ├─ 昨天学的新词
  ├─ 高频错误词
  └─ Boss关词汇
```

### 6.2 考试模式

**设计理念**：分离游戏体验和考试训练

```
考试模式界面：
  ┌─────────────────────────────────────┐
  │  🎮 游戏模式        📝 考试模式    │
  └─────────────────────────────────────┘

考试模式包含：

1. 听力训练（15分钟）
   ├─ 听音选图
   ├─ 听音选词
   └─ 对话理解

2. 阅读理解（15分钟）
   ├─ 短文阅读
   ├─ 完形填空
   └─ 信息匹配

3. 写作训练（10分钟）
   ├─ 看图写词
   ├─ 连词成句
   └─ 简单改错

4. 模拟测试（20分钟）
   └─ KET真题风格
```

**成绩报告**：

```
每周生成报告：
  ├─ 词汇掌握数：125/300
  ├─ 正确率趋势图
  ├─ 薄弱环节：look vs see
  ├─ 学习时长：本周5.2小时
  └─ 对比KET要求：达标词汇42%
```

## 八、技术架构

### 8.1 技术栈选型

```
前端（建议二选一）：

选项1：Flutter
  ├─ 优点：跨平台、性能好、UI灵活
  ├─ 缺点：团队可能不熟悉
  └─ 适合：快速迭代、多端统一

  ├─ 缺点：包体大
  └─ 适合：重度游戏化、复杂动画

```

## 九、MVP开发计划

### 9.1 MVP目标

**核心假设**：
> 如果10岁孩子在游戏中通过4种玩法学习词汇，并有智能复习系统保障，
> 那么他们能在1个月内掌握150个KET核心词汇，且次日留存率>50%

**验证指标**：
- 第2天复测正确率 ≥ 70%
- 单次游戏时长 ≥ 8分钟
- 次日留存率 ≥ 50%
- 家长满意度 ≥ 4.0/5.0

### 9.2 MVP范围

**包含内容**：
- ✅ 2.5个主题世界（LOOK、MOVE、SAY前半）
- ✅ 4种游戏玩法
- ✅ 智能复习系统
- ✅ 家长报告基础版
- ✅ 约180个词汇
- ✅ 约35个关卡

**不包含内容**：
- ❌ 完整8个主题
- ❌ 词根超能力系统
- ❌ 社交功能
- ❌ PET词汇
- ❌ 完整考试模式
- ❌ 付费系统（免费体验）

### 9.3 12周开发排期

```
第1-2周：基础搭建 + 设计定稿
  ├─ 确定技术栈（Flutter/Unity）
  ├─ 搭建项目框架
  ├─ 数据库设计
  ├─ UI设计稿定稿
  └─ 词汇表最终确定

第3-5周：核心玩法开发
  ├─ 4种玩法机制实现
  ├─ 场景系统
  ├─ 反馈系统（三级提示）
  └─ 奖励系统基础版

第6-8周：内容制作
  ├─ 2.5个世界关卡开发
  ├─ 180个词汇数据录入
  ├─ 音频录制/采购
  ├─ 场景美术制作
  └─ UI动效

第9-10周：学习系统 + 数据
  ├─ 智能复习算法
  ├─ 学习进度追踪
  ├─ 家长报告（简版）
  └─ 数据埋点

第11周：测试调优
  ├─ 内部功能测试
  ├─ 5-10名10岁儿童试玩测试
  ├─ 难度调整
  ├─ Bug修复

第12周：发布准备
  ├─ App Store/Google Play准备
  ├─ 宣传材料
  └─ 软发/灰发
```

## 十、数据指标与验收

### 10.1 北极星指标

**定义**：
> 有效学习词汇数 = 孩子在游戏中学会且能正确使用的词汇数量

**计算方式**：
- 首次正确 + 次日复测正确 = 掌握

**目标**：
- MVP阶段：每人平均掌握120词（180的67%）

### 10.2 分层指标体系

```
第一层：产品健康度（最重要）
  ├─ 次日留存 ≥ 50%
  ├─ 7日留存 ≥ 30%
  └─ 月留存 ≥ 15%

第二层：学习效果（核心价值）
  ├─ 首次学习正确率 ≥ 75%
  ├─ 次日复测正确率 ≥ 70%
  ├─ 7日后正确率 ≥ 60%
  └─ 月平均掌握词汇 ≥ 100词

第三层：参与度
  ├─ DAU/MAU ≥ 0.25（月活中25%日活）
  ├─ 单次时长 ≥ 8分钟
  ├─ 周使用次数 ≥ 4次
  └─ 关卡完成率 ≥ 60%

第四层：付费（MVP后）
  ├─ 免费转付费率 ≥ 8%
  ├─ ARPU ≥ $8/月
  └─ 12个月LTV ≥ $60

第五层：家长满意度
  ├─ App Store评分 ≥ 4.3
  ├─ 家长推荐度 ≥ 70%
  └─ 客服投诉率 ≤ 2%
```

### 10.3 A/B测试计划

**测试1：玩法偏好**
- A组：以"听音寻宝"为主（50%）
- B组：以"词句配对"为主（50%）
- 指标：留存率、单次时长

**测试2：复习频率**
- A组：每天复习10词
- B组：每3天复习30词
- 指标：记忆保持率

**测试3：反馈方式**
- A组：错误后立即提示
- B组：做完一题再反馈
- 指标：正确率、挫败感评分

## 十一、风险与应对

### 11.1 产品风险

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| 孩子觉得不好玩 | 高 | 高 | 前2周多次儿童试玩，及时调整 |
| 学习效果不明显 | 中 | 高 | 对比组实验、强化复习系统 |
| 词根概念太抽象 | 高 | 中 | 三阶段渐进、降低初期权重 |
| 关卡难度失衡 | 中 | 中 | 动态调整算法、手动平衡 |

### 11.2 市场风险

| 风险 | 应对 |
|------|------|
| 竞品抄袭 | 快速建立品牌、内容壁垒 |
| 获客成本高 | 口碑传播、家长社区 |
| 政策变化 | 关注教育政策、合规运营 |

### 11.3 技术风险

| 风险 | 应对 |
|------|------|
| 性能问题 | 充分测试、资源优化 |
| 数据丢失 | 云端备份、本地缓存 |
| 兼容性问题 | 多机型测试 |

```
  ├─ 基础复习功能
```
